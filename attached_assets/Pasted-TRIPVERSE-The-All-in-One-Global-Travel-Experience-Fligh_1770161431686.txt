2.1 System Architecture (Microservices)
Service	Purpose
Auth Service	Login, OTP, JWT
KYC Service	ID verification, face match
User Service	Profiles, roles
Listings Service	Stays, Companions, Safaris, Restaurants
Booking Service	Reservations, availability
Chat Service	Real-time messaging
Payment Service	Escrow, payouts
Review Service	Ratings
Safety Service	Panic, reports
Admin Service	Moderation, audits

All services communicate via REST + events (Kafka/RabbitMQ).

2.3 Database Schema (PostgreSQL)
USERS(id PK, role, name, email, phone, country, verified, created_at)

KYC(user_id FK, id_type, id_number, status)

STAYS(id PK, host_id FK, city, price, status)
COMPANIONS(id PK, user_id FK, city, rate, status)
SAFARIS(id PK, provider_id FK, city, price, license_verified)
RESTAURANTS(id PK, name, city)

BOOKINGS(id PK, user_id FK, type, reference_id, start_time, end_time, status)

PAYMENTS(id PK, user_id, amount, status, escrow)

CHATS(id PK, type, reference_id)
MESSAGES(id PK, chat_id FK, sender_id, message)

REVIEWS(id PK, reviewer_id, target_id, rating, comment)

SAFETY_REPORTS(id PK, reporter_id, target_id, reason)
